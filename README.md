# GoblinMaster
GoblinMaster es una aplicación web desarrollada como proyecto final de la Tecnicatura Universitaria en Programación (UTN - Facultad Regional Mendoza). Su objetivo es brindar una herramienta integral para Dungeon Masters (directores de juego) y jugadores en partidas de rol de mesa, especialmente aquellas basadas en el sistema Pathfinder.  El backend, desarrollado en Java con Spring Boot, se encarga de gestionar la lógica principal del sistema, la autenticación de usuarios, y la comunicación con las bases de datos. 
La arquitectura se diseñó para combinar MySQL (para datos estructurados como usuarios, personajes y combates) y MongoDB (para información más flexible, como bestiarios, dotes y reglas).

#Tecnologías utilizadas
--Java 21
--Spring Boot 3.5.6
--Spring Data JPA (MySQL)
--Spring Data MongoDB (MongoDB)
--Spring Security + JWT
--Lombok, ModelMapper, Validation API
--Maven para gestión de dependencias

#Objetivos
--Facilitar la gestión de partidas de rol de mesa para Dungeon Masters.
--Centralizar la información de combate, personajes, NPCs y reglas del sistema.
--Permitir la creación de campañas y encuentros personalizables.
--Proveer una API segura, modular y escalable, con endpoints RESTful listos para integrarse con el frontend desarrollado en React + TypeScript.

#Contexto académico
Este proyecto forma parte del trabajo final de la Tecnicatura Universitaria en Programación (UTN-FRM), con el propósito de integrar los conocimientos adquiridos a lo largo de la carrera en desarrollo web full-stack, arquitectura de software y bases de datos.

#GoblinMaster
GoblinMaster is a web application developed as a final project for the University Degree in Programming (UTN - Mendoza Regional Faculty). Its objective is to provide a comprehensive tool for Dungeon Masters (game masters) and players in tabletop role-playing games, especially those based on the Pathfinder system. The backend, developed in Java with Spring Boot, is responsible for managing the system's main logic, user authentication, and communication with databases.
The architecture was designed to combine MySQL (for structured data such as users, characters, and combats) and MongoDB (for more flexible information such as bestiaries, feats, and rules).

#Technologies Used
--Java 21
--Spring Boot 3.5.6
--Spring Data JPA (MySQL)
--Spring Data MongoDB (MongoDB)
--Spring Security + JWT
--Lombok, ModelMapper, Validation API
--Maven for dependency management

#Objectives
--To facilitate the management of tabletop role-playing games for Dungeon Masters.
--To centralize information on combat, characters, NPCs, and system rules.
--To allow the creation of customizable campaigns and encounters.
--To provide a secure, modular, and scalable API with RESTful endpoints ready to integrate with the frontend developed in React + TypeScript.

#Academic Context
This project is part of the final project for the University Degree in Programming (UTN-FRM), with the goal of integrating the knowledge acquired throughout the program in full-stack web development, software architecture, and databases.
